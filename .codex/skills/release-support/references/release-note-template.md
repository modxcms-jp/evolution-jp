# Release Note Template

以下を下書きに追記して最終版へ整える。一次情報の優先順位は「PR本文/差分 > コミット件名」。

## 1. タイトル

- 形式例: `Evolution CMS JP Edition 1.2.1J リリース`

## 2. 概要（3行以内）

- 何が改善されたか
- 利用者への影響
- アップグレード推奨有無

## 3. 主な変更

- `feat` / `fix` / `refactor` の順に重要項目を3-7件へ圧縮する。
- 内部変更のみの場合は運用影響の有無を明示する。

## 4. Pull Request

- スクリプト出力の PR 一覧から主要 PR を選び、1行コメントを足す。

## 5. アップグレード時の注意

- 互換性影響
- 設定変更の有無
- キャッシュクリア要否

## 6. 配布情報

- GitHub Release URL
- zip 資材名
